# 技能漏洞检测指南

对照报错检查 skill 的 SKILL.md，找三类漏洞。

## 漏洞类型

### 1. 规则缺失
skill 说了"要做什么"，但没说"什么情况下不能做"。

检测方法：
- 找到 skill 中给出指令的段落
- 看这个指令有没有标注前置条件或禁止条件
- 对照报错：如果报错是因为"在某种情况下执行了指令"，那就是规则缺失

示例：
```
skill 说："用 PyInstaller 打包"
报错：路径含中文导致 script not found
漏洞：缺"打包前确保路径只用 ASCII"
```

### 2. 边界遗漏
skill 覆盖了核心路径，但没提边界条件。

检测方法：
- 找到 skill 描述核心流程的段落
- 看有没有"如果失败怎么办"、"特殊情况"段落
- 对照报错：报错是否发生在边界条件（空值/超时/大文件/特殊字符）

示例：
```
skill 说："安装脚本用 bat"
报错：bat 中文乱码
漏洞：缺"bat 不能含中文，用 ASCII 或换 VBS"
```

### 3. 检查缺失
skill 说"完成后验证"，但没说具体验证什么。

检测方法：
- 找到 skill 中提到"验证"、"检查"、"测试"的段落
- 看验证步骤是否具体（有具体命令 vs 只有"确保正确"）
- 对照报错：报错是否在"验证通过"之后仍然发生

示例：
```
skill 说："测试通过后提交"
报错：测试过了但 exe 是旧版本
漏洞：缺"测试后必须重建 exe 再验证"
```

## 输出格式

```
### 技能漏洞：skill-name
- **类型**：规则缺失 / 边界遗漏 / 检查缺失
- **位置**：SKILL.md 第 X-Y 行
- **触发错误**：[错误 #N]
- **建议**：[具体修改建议]
```
