# skill-optimizer — AI Agent Skill 优化器

分析对话上下文、按置信度识别 skill 改进点、经长期复利滤网过滤、用户确认后执行修改的元技能。

## 为什么需要

- skill 用久了会腐烂：触发词不准、流程冗余、内容过时
- 改 skill 全靠直觉，没有系统化的分析框架
- 盲目修改带来长期副作用——为今天的对话加补丁，明天就后悔

skill-optimizer 提供一套完整的分析→分级→滤网→执行管线，确保每次 skill 修改都是长期复利。

## 功能

1. **上下文采集** — 分析对话中 skill 的触发/执行/纠正事件，条件查询历史记忆
2. **条件复盘** — 有异常信号时自动调用 [debug-architect](../debug-architect/) 获取硬证据
3. **三维并行审查** — 触发准确度 + 内容质量 + 工作流效率，三个 agent 并行
4. **置信度分级** — 来源信号 × 证据强度 × 影响范围 → [高]/[中]/[低] 三档
5. **对抗验证** — 中等置信度发现由独立 agent 反向论证，确认后才升级
6. **长期复利滤网** — 三问（系统性？庆幸？规则vs补丁？）+ 五条红线，拦截短期补丁
7. **用户决策** — 结构化报告，逐条/批量确认
8. **安全执行** — 修改后自检（YAML/Markdown/触发词交叉），失败自动回滚

## 依赖

- **可选：** [debug-architect](../debug-architect/) — 作为硬证据引擎，不可用时自动降级为对话推断

## 设计理念

- **最小改动，最大收益** — 每条改动可追溯到具体对话问题
- **长期复利优先** — 改规则不补补丁，三问滤网 + 五条红线
- **置信度驱动** — 不确定的不动，中等确定的不验证不动
- **优雅降级** — 依赖不可用时自动切换路径
